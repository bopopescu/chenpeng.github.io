前缀树Trie router

以前学数据结构与算法的时候，说到解析路由可以使用Trie树。Trie (发音为 “try”) 或前缀树是一种树数据结构，用于检索字符串数据集中的键。这一高效的数据结构有多种应用：

  1. 自动补全
  2. 拼写检查
  3. IP路由（最长前缀匹配）

上面的router使用的是map，对于动态路由没法支持，比如常见的/user/1，1是id。

trie树搞定，开始接到router里。在restful风格的接口定义中，因为不同的请求方法，相同的url表示的功能不一样。所以，我们的router里需要一个trie数组，支持“GET”，“POST”，“PUT”，“DELETE”等请求方法。

如上图所示，这棵树包含了/user/:id/address,/user/:id/age,/product3个路由。其中，“:id”这种用黄色标记，表示动态路由。末节点用红色标记。

通过Trie树插入路由，查询路由的时候会把类似/user/1/address转成/user/:id/address，通过这个key去map里查询到相应的handler。
此时，忽然发现，还没有方法把“:id”具体的值取出来。context中新增Params，用来获取:id的值。
